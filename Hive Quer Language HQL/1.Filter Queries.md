# Filtering Queries in Hive on Complex Data Types

## Introduction

Filtering is one of the most commonly used operations in Hive. It allows users to retrieve only the records that satisfy specific conditions instead of scanning and displaying the entire dataset.

Hive supports filtering not only on primitive data types such as INT, STRING, and DATE, but also on complex data types such as:

* MAP
* STRUCT
* ARRAY

This makes Hive extremely powerful for analyzing semi-structured data.

The employee table used in this chapter contains all three complex data types, making it an excellent example for understanding how filtering works in Hive.

---

# Understanding the Employee Table Structure

Before writing queries, it is important to understand the structure of the table.

```text
Employee Table
│
├── id
│
├── name (MAP)
│     ├── firstname
│     └── lastname
│
├── info (STRUCT)
│     ├── birth_date
│     └── gender
│
├── job (STRUCT)
│     ├── hire_date
│     ├── dept_no
│     ├── dept_name
│     └── salary
│
└── dept_history (ARRAY)
```

Since the table contains nested structures, different techniques are required to access each type of data.

---

# Filtering Data in MAP Columns

## MAP Data Type Recap

A MAP stores data as key-value pairs.

Example:

```text
name
│
├── firstname → Georgie
└── lastname  → Facello
```

In Hive, values inside a MAP are accessed using square bracket notation.

Syntax:

```sql
map_column['key']
```

---

## Finding Employees with First Name Georgie

Query:

```sql
SELECT id
FROM employees
WHERE name['firstname'] = 'Georgie';
```

### Explanation

The query performs the following operations:

1. Accesses the firstname key from the name MAP.
2. Compares it with "Georgie".
3. Returns employee IDs of matching records.

Visualization:

```text
name
│
├── firstname → Georgie ✔
└── lastname  → Facello
```

Only matching records are returned.

Result:

```text
253 Employees
```

---

# Applying Multiple Conditions on MAP Data

Filtering often requires multiple conditions.

Suppose we want employees whose:

* First name = Georgie
* Last name = Facello

Query:

```sql
SELECT id
FROM employees
WHERE name['firstname']='Georgie'
AND name['lastname']='Facello';
```

---

## Why Use AND?

The AND operator requires both conditions to be true.

```text
Condition 1 = TRUE
AND
Condition 2 = TRUE
```

Only then is the row selected.

Result:

```text
2 Employees
```

This significantly narrows the search.

---

# Filtering Data in STRUCT Columns

## STRUCT Data Type Recap

A STRUCT groups related fields together.

Example:

```text
job
│
├── hire_date
├── dept_no
├── dept_name
└── salary
```

Fields inside a STRUCT are accessed using the dot operator.

Syntax:

```sql
struct_column.field_name
```

---

# Counting Employees in Marketing Department

Query:

```sql
SELECT COUNT(*)
FROM employees
WHERE job.dept_name='Marketing';
```

---

## Explanation

The query:

1. Reads the job STRUCT.
2. Extracts dept_name.
3. Filters rows where department equals Marketing.
4. Counts matching records.

Visualization:

```text
job
│
├── dept_no
├── dept_name → Marketing ✔
└── salary
```

Result:

```text
≈ 18,500 Employees
```

---

## COUNT Function

COUNT() is an aggregate function used to determine the number of records satisfying a condition.

Syntax:

```sql
COUNT(*)
```

The asterisk (*) means count all matching rows.

---

# MapReduce Execution During Aggregation

Simple filtering queries may not always launch a MapReduce job.

However:

```sql
COUNT(*)
```

requires aggregation.

Hive therefore generates a MapReduce job.

Process:

```text
Records
   ↓
Mapper
   ↓
Partial Counts
   ↓
Reducer
   ↓
Final Count
```

This is why counting operations often take longer than simple selections.

---

# Filtering Multiple Values Using IN

Suppose we want employees belonging to:

* Marketing
* Finance

Writing:

```sql
dept='Marketing'
OR dept='Finance'
```

works, but Hive provides a cleaner solution.

---

## IN Clause

Syntax:

```sql
column IN (value1,value2,...)
```

---

## Query

```sql
SELECT COUNT(*)
FROM employees
WHERE job.dept_name
IN ('Marketing','Finance');
```

---

## Internal Logic

Equivalent to:

```sql
job.dept_name='Marketing'
OR
job.dept_name='Finance'
```

---

## Why Not Use AND?

This would be incorrect:

```sql
dept='Marketing'
AND
dept='Finance'
```

because one employee cannot belong to both departments simultaneously.

---

Result:

```text
≈ 34,000 Employees
```

---

# Working with Date Functions

## Date Fields in STRUCT

The employee table contains:

```text
info
│
├── birth_date
└── gender
```

The birth_date field is stored as a DATE type.

Example:

```text
1965-04-23
```

---

# Extracting the Year

Hive provides built-in date functions.

One such function is:

```sql
YEAR()
```

Syntax:

```sql
YEAR(date_column)
```

Returns only the year component.

Example:

```sql
YEAR('1965-04-23')
```

Output:

```text
1965
```

---

# Employees Born After 1960

Query:

```sql
SELECT COUNT(*)
FROM employees
WHERE YEAR(info.birth_date) > 1960;
```

---

## Explanation

Step 1:

Access birth_date field.

```sql
info.birth_date
```

Step 2:

Extract year.

```sql
YEAR(info.birth_date)
```

Step 3:

Apply filter.

```sql
> 1960
```

Step 4:

Count matching employees.

Result:

```text
≈ 94,000 Employees
```

---

# Filtering Data in ARRAY Columns

## ARRAY Data Type Recap

An ARRAY stores multiple values in sequence.

Example:

```text
dept_history
│
├── D001
├── D002
└── D003
```

Each element has an index.

```text
Index 0 → D001
Index 1 → D002
Index 2 → D003
```

---

# Accessing ARRAY Elements

Syntax:

```sql
array_column[index]
```

Example:

```sql
dept_history[0]
```

returns first element.

---

# Finding Department Transfers

Suppose we want employees who moved from:

```text
D002
   ↓
D003
```

Query:

```sql
SELECT COUNT(*)
FROM employees
WHERE dept_history[0]='D002'
AND dept_history[1]='D003';
```

---

## Explanation

Hive checks:

First Department:

```text
D002
```

Second Department:

```text
D003
```

If both conditions are satisfied, the employee is counted.

Result:

```text
≈ 1,800 Employees
```

---

# Logical Operators in Filtering

## AND Operator

Returns rows only when all conditions are true.

Example:

```sql
firstname='Georgie'
AND
lastname='Facello'
```

---

## OR Operator

Returns rows when at least one condition is true.

Example:

```sql
dept='Marketing'
OR
dept='Finance'
```

---

## IN Operator

Shorthand for multiple OR conditions.

Example:

```sql
dept IN ('Marketing','Finance')
```

---

# Access Operators Summary

| Data Type | Access Method |
| --------- | ------------- |
| MAP       | map['key']    |
| STRUCT    | struct.field  |
| ARRAY     | array[index]  |

Examples:

```sql
name['firstname']
```

```sql
job.dept_name
```

```sql
dept_history[0]
```

These three access patterns are among the most important concepts when working with Hive complex data types.

# Quick Revision

* Filtering in Hive is performed using the WHERE clause.
* MAP values are accessed using square brackets.
* STRUCT fields are accessed using the dot operator.
* ARRAY elements are accessed using indexes.
* AND requires all conditions to be true.
* OR requires at least one condition to be true.
* IN simplifies multiple OR conditions.
* YEAR() extracts the year from a DATE.
* COUNT(*) performs aggregation and may trigger a MapReduce job.

# One-Line Recall

**Hive filtering queries can operate on MAP, STRUCT, ARRAY, and DATE fields using specialized access operators, enabling powerful analysis of complex and semi-structured data.**

# More detailed answer

**Hive filter queries use the WHERE clause to retrieve records based on conditions. Hive supports filtering on primitive as well as complex data types such as MAP, STRUCT, and ARRAY. MAP values are accessed using map['key'], STRUCT fields are accessed using struct.field, and ARRAY elements are accessed using indexes starting from 0. Multiple conditions can be combined using AND, OR, and IN clauses. Hive also provides date functions such as YEAR() for filtering records based on dates. Aggregate queries like COUNT(*) often execute as MapReduce jobs when processing large datasets. These features make Hive powerful for analyzing structured and semi-structured data stored in Hadoop.**
