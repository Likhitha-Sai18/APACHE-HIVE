# Problem 1

## Tables

### users

| user_id | name  |
| ------- | ----- |
| 1       | John  |
| 2       | Alice |
| 3       | Bob   |

### orders

| order_id | user_id |
| -------- | ------- |
| 101      | 1       |
| 102      | 2       |
| 103      | 1       |

## Question

Display customer names and their order IDs.

## Hive Query

```sql
SELECT u.name,
       o.order_id
FROM users u
INNER JOIN orders o
ON u.user_id = o.user_id;
```

## Explanation

* INNER JOIN returns matching records only.
* Join key = `user_id`.
* Users without orders are excluded.

# Problem 31 — Subquery + JOIN

## Question

Find customers whose total purchase amount is greater than the average customer purchase amount.

## Hive Query

```sql id="cvys4g"
SELECT u.name,
       t.total_amount
FROM users u
JOIN (
    SELECT user_id,
           SUM(amount) AS total_amount
    FROM orders
    GROUP BY user_id
) t
ON u.user_id = t.user_id
WHERE t.total_amount >
(
    SELECT AVG(total_sales)
    FROM (
        SELECT SUM(amount) AS total_sales
        FROM orders
        GROUP BY user_id
    ) x
);
```

## Explanation

* Inner subquery calculates customer totals.
* Second subquery calculates average total.
* Final query compares both.

```sql id="pwgv76"
ON u.user_id = o.user_id
GROUP BY u.name
ORDER BY revenue DESC
LIMIT 1;
```

## Explanation

Very common Data Analyst / Data Engineer question.

# Understanding SELF JOIN

We use the same table twice.

```sql id="8mtu3r"
employees e
JOIN employees m
```

Think:

```text id="l4sl3t"
e = Employee

m = Manager
```

Same table, different roles.

# Problem 41 — Employee and Manager Name

## Question

Display employee name and manager name.

## Hive Query

```sql id="5slyzv"
SELECT e.emp_name AS employee,
       m.emp_name AS manager
FROM employees e
LEFT JOIN employees m
ON e.manager_id = m.emp_id;
```

## Explanation

* Employee points to manager through `manager_id`.
* Manager record is found using `emp_id`.

# Problem 42 — Employees Working Under John

## Question

Find all employees reporting directly to John.

## Hive Query

```sql id="6zg53j"
SELECT e.emp_name
FROM employees e
JOIN employees m
ON e.manager_id = m.emp_id
WHERE m.emp_name = 'John';
```

## Explanation

* Find John's employee record.
* Match employees whose manager_id = John's emp_id.

# Problem 45 — Managers Having More Than 1 Employee

## Question

Find managers supervising more than one employee.

## Hive Query

