# Problem 61 — Customers Above Average Spending

## Question

Find customers whose total spending is greater than the average customer spending.

## Hive Query

```sql
SELECT u.name,
       t.total_spending
FROM users u
JOIN (
    SELECT user_id,
           SUM(amount) AS total_spending
    FROM orders
    GROUP BY user_id
) t
ON u.user_id = t.user_id
WHERE t.total_spending >
(
    SELECT AVG(customer_total)
    FROM (
        SELECT SUM(amount) AS customer_total
        FROM orders
        GROUP BY user_id
    ) x
);
```

## Explanation

* First calculate spending per customer.
* Calculate average spending.
* Compare both.

# Problem 71 — Small Table + Large Table Join

## Scenario

### users

```text
80 KB
```

### transactions

```text
500 GB
```

## Question

Join users and transactions efficiently.

## Basic Hive Query

```sql id="r13j8g"
SELECT u.user_name,
       t.transaction_amount
FROM users u
JOIN transactions t
ON u.user_id = t.user_id;
```

## Optimization

```sql id="5t7n3m"
SET hive.auto.convert.join=true;
```

## Explanation

* Hive detects that `users` is very small.
* Converts join into a **Map-Side Join**.
* Avoids reducer phase.

### Interview Point

This is one of the most asked Hive optimization questions.

# Problem 91 — Most Valuable Customer

## Tables

### users

| user_id | name  |
| ------- | ----- |

| 2       | Alice |
| 3       | Bob   |

### orders

| order_id | user_id | amount |

## Question

Find the customer generating the highest revenue.

## Hive Query

```sql
SELECT u.name,
       SUM(o.amount) AS revenue
FROM users u
```

* Sort descending.
* Pick top customer.

# Problem 95 — Above Average Revenue Customer

## Question

Find customers spending above average customer spending.

## Hive Query

```sql
SELECT u.name,
       SUM(o.amount) AS revenue
FROM users u
JOIN orders o
ON u.user_id = o.user_id
GROUP BY u.name
HAVING SUM(o.amount) >
(
   SELECT AVG(total_sales)
   FROM
   (
      SELECT SUM(amount) AS total_sales
      FROM orders
      GROUP BY user_id
   ) t
);
```

## Explanation

JOIN + GROUP BY + HAVING + Subquery
