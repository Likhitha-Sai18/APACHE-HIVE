# Joins in Hive

## Introduction

In real-world databases and data warehouses, information is often stored across multiple tables instead of a single large table. To retrieve related information from these tables, we use **Joins**.

A Join combines rows from two or more tables based on a related column known as the **Join Key**.

For example:

### Employees Table

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |
| 103    | Bob   |

### Departments Table

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |
| 104    | Finance    |

Suppose we want:

| Name  | Department |
| ----- | ---------- |
| John  | IT         |
| Alice | HR         |

The information exists in two different tables. A Join helps us combine them.

---

# Why Do We Need Joins?

Without joins:

```text
Employee Information
     +
Department Information
```

cannot be combined efficiently.

Joins help:

* Combine related data
* Perform analysis across multiple tables
* Avoid data duplication
* Maintain normalized database structures

---

# Join Key

A Join is performed on a common column present in both tables.

Example:

### Employees

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |

### Departments

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |

Common Column:

```text
Emp_ID
```

This becomes the Join Key.

---

# Types of Joins in Hive

```text
Joins
│
├── Inner Join
├── Left Outer Join
├── Right Outer Join
└── Full Outer Join
```

---

# 1. INNER JOIN

## Definition

An Inner Join returns only the records that exist in both tables.

Only matching rows are included.

---

## Visualization

### Employees

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |
| 103    | Bob   |

### Departments

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |
| 104    | Finance    |

Common IDs:

```text
101
102
```

---

## Query

```sql
SELECT *
FROM Employees e
INNER JOIN Departments d
ON e.Emp_ID = d.Emp_ID;
```

---

## Result

| Emp_ID | Name  | Department |
| ------ | ----- | ---------- |
| 101    | John  | IT         |
| 102    | Alice | HR         |

---

## Why Bob Is Missing?

Bob has:

```text
Emp_ID = 103
```

but there is no matching record in Departments.

Therefore Inner Join excludes him.

---

## Memory Trick

```text
INNER = INTERSECTION
```

Only common data.

---

# 2. LEFT OUTER JOIN

## Definition

A Left Join returns:

* All rows from the Left Table
* Matching rows from the Right Table

If no match exists, NULL values are returned.

---

## Visualization

### Employees (Left Table)

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |
| 103    | Bob   |

### Departments (Right Table)

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |

---

## Query

```sql
SELECT *
FROM Employees e
LEFT JOIN Departments d
ON e.Emp_ID = d.Emp_ID;
```

---

## Result

| Emp_ID | Name  | Department |
| ------ | ----- | ---------- |
| 101    | John  | IT         |
| 102    | Alice | HR         |
| 103    | Bob   | NULL       |

---

## Explanation

Bob exists in Employees but not in Departments.

Therefore:

```text
Department = NULL
```

---

## Visualization

```text
LEFT TABLE
  101 ✔
  102 ✔
  103 ✔

RIGHT TABLE
  101 ✔
  102 ✔

Output:
101
102
103
```

All records from the left table survive.

---

## Memory Trick

```text
LEFT JOIN
=
KEEP EVERYTHING FROM LEFT
```

---

# 3. RIGHT OUTER JOIN

## Definition

A Right Join returns:

* All rows from the Right Table
* Matching rows from the Left Table

If no match exists, NULL values appear on the left side.

---

## Example

### Employees

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |

### Departments

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |
| 104    | Finance    |

---

## Query

```sql
SELECT *
FROM Employees e
RIGHT JOIN Departments d
ON e.Emp_ID = d.Emp_ID;
```

---

## Result

| Emp_ID | Name  | Department |
| ------ | ----- | ---------- |
| 101    | John  | IT         |
| 102    | Alice | HR         |
| 104    | NULL  | Finance    |

---

## Explanation

Department Finance exists.

Employee information does not.

Therefore:

```text
Name = NULL
```

---

## Memory Trick

```text
RIGHT JOIN
=
KEEP EVERYTHING FROM RIGHT
```

---

# Difference Between LEFT and RIGHT JOIN

### LEFT JOIN

```text
Left Table → Complete
Right Table → Matching Only
```

---

### RIGHT JOIN

```text
Right Table → Complete
Left Table → Matching Only
```

---

# 4. FULL OUTER JOIN

## Definition

A Full Outer Join returns:

* All rows from Left Table
* All rows from Right Table

Matched rows are merged.

Unmatched rows contain NULL values.

---

## Example

### Employees

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |
| 103    | Bob   |

### Departments

| Emp_ID | Department |
| ------ | ---------- |
| 101    | IT         |
| 102    | HR         |
| 104    | Finance    |

---

## Query

```sql
SELECT *
FROM Employees e
FULL OUTER JOIN Departments d
ON e.Emp_ID = d.Emp_ID;
```

---

## Result

| Emp_ID | Name  | Department |
| ------ | ----- | ---------- |
| 101    | John  | IT         |
| 102    | Alice | HR         |
| 103    | Bob   | NULL       |
| 104    | NULL  | Finance    |

---

## Explanation

Included:

### Common Records

```text
101
102
```

### Left-only Record

```text
103
```

### Right-only Record

```text
104
```

---

## Visualization

```text
Employees          Departments

101 ✔             101 ✔
102 ✔             102 ✔
103 ✔
                  104 ✔

FULL JOIN

101
102
103
104
```

---

## Memory Trick

```text
FULL OUTER JOIN
=
LEFT JOIN + RIGHT JOIN
```

| 3  |

```text

```

## LEFT JOIN


---

## RIGHT JOIN

```text
2
```

# Hive Join Syntax

### Inner Join

```sql
SELECT *
FROM table1 t1
INNER JOIN table2 t2
ON t1.id = t2.id;
```

---

### Left Join

```sql
SELECT *
FROM table1 t1
LEFT OUTER JOIN table2 t2
ON t1.id = t2.id;
```

---

### Right Join

```sql
SELECT *
FROM table1 t1
RIGHT OUTER JOIN table2 t2
ON t1.id = t2.id;
```

---

### Full Outer Join

```sql
SELECT *
FROM table1 t1
FULL OUTER JOIN table2 t2
ON t1.id = t2.id;
```

# Real-World Example

### Employee Table

| Emp_ID | Name  |
| ------ | ----- |
| 101    | John  |
| 102    | Alice |

### Salary Table

| Emp_ID | Salary |
| ------ | ------ |
| 101    | 50000  |
| 102    | 70000  |

Inner Join:

```sql
SELECT Name, Salary
FROM Employee e
JOIN Salary s
ON e.Emp_ID = s.Emp_ID;
```

Output:

| Name  | Salary |
| ----- | ------ |
| John  | 50000  |
| Alice | 70000  |

This is one of the most common joins used in Data Engineering.

# Interview Questions

### What is a Join?

A Join combines rows from multiple tables using a common column.

---

### Which Join returns only matching rows?

```text
INNER JOIN
```

---

### Which Join returns all rows from the left table?

```text
LEFT JOIN
```

---

### Which Join returns all rows from the right table?

```text
RIGHT JOIN
```

---

### Which Join returns all rows from both tables?

```text
FULL OUTER JOIN
```

---

### What is a Join Key?

The common column used to connect two tables.

Example:

```text
Emp_ID
```

---

# Quick Revision

```text
INNER JOIN  → Common Records Only

LEFT JOIN   → All Left + Matching Right

RIGHT JOIN  → All Right + Matching Left

FULL JOIN   → All Records from Both Tables
```

---

# One-Line Recall

**Joins combine related data from multiple tables using a common key, with Inner Join returning common records, Left/Right Join preserving one table completely, and Full Outer Join preserving both tables completely.**
