# Left Join, Finding Missing Records, and Full Outer Join in Hive

This topic covers one of the most important real-world uses of joins:

> Finding records that exist in one table but do not exist in another table.

This pattern appears very frequently in interviews, SQL tests, Hive projects, and Data Engineering jobs.

---

# Problem Scenario

We have two tables:

### Users Table

Contains all registered users.

| user_id | name  | country |
| ------- | ----- | ------- |
| 101     | John  | India   |
| 102     | Alice | USA     |
| 103     | Bob   | India   |
| 104     | David | UK      |

---

### Submissions Table

Contains users who submitted solutions.

| user_id | problem_id |
| ------- | ---------- |
| 101     | P101       |
| 102     | P102       |

Notice:

```text
103 → No Submission
104 → No Submission
```

These users registered but never submitted any solution.

---

# Why INNER JOIN Cannot Solve This

Suppose we run:

```sql
SELECT *
FROM users u
INNER JOIN submissions s
ON u.user_id = s.user_id;
```

Result:

| user_id | name  | problem_id |
| ------- | ----- | ---------- |
| 101     | John  | P101       |
| 102     | Alice | P102       |

Notice:

```text
103 Missing
104 Missing
```

Why?

Because INNER JOIN only keeps matching records.

---

# Understanding LEFT JOIN

A LEFT JOIN keeps:

```text
ALL Records from Left Table
+
Matching Records from Right Table
```

---

## Query

```sql
SELECT *
FROM users u
LEFT JOIN submissions s
ON u.user_id = s.user_id;
```

---

## Result

| user_id | name  | problem_id |
| ------- | ----- | ---------- |
| 101     | John  | P101       |
| 102     | Alice | P102       |
| 103     | Bob   | NULL       |
| 104     | David | NULL       |

---

Notice:

```text
Bob     → NULL
David   → NULL
```

The NULL values indicate:

> No matching submission exists.

---

# Finding Users Who Never Submitted Any Problem

This is where LEFT JOIN becomes extremely useful.

The logic is:

### Step 1

Return all users.

```sql
LEFT JOIN
```

### Step 2

Keep only rows where submission information is missing.

```sql
WHERE problem_id IS NULL
```

---

## Query

```sql
SELECT *
FROM users u
LEFT JOIN submissions s
ON u.user_id = s.user_id
WHERE s.problem_id IS NULL;
```

---

## Output

| user_id | name  |
| ------- | ----- |
| 103     | Bob   |
| 104     | David |

These are exactly the users who never submitted any solution.

---

# Why Does This Work?

After LEFT JOIN:

| user_id | problem_id |
| ------- | ---------- |
| 101     | P101       |
| 102     | P102       |
| 103     | NULL       |
| 104     | NULL       |

Applying:

```sql
WHERE problem_id IS NULL
```

keeps only:

| user_id | problem_id |
| ------- | ---------- |
| 103     | NULL       |
| 104     | NULL       |

---

# Country-wise Count of Users With No Submissions

Suppose management asks:

> Which countries have the most inactive users?

---

## Query

```sql
SELECT
    country,
    COUNT(*)
FROM users u
LEFT JOIN submissions s
ON u.user_id = s.user_id
WHERE s.problem_id IS NULL
GROUP BY country
ORDER BY COUNT(*) DESC;
```

---

### Example Output

| Country | Users |
| ------- | ----- |
| India   | 500   |
| USA     | 350   |
| UK      | 100   |

This shows countries where users register but never participate.

---

# Most Common Interview Pattern

Whenever you hear:

```text
Find records present in A but not in B
```

Think:

```sql
LEFT JOIN
+
WHERE right_table.column IS NULL
```

---

### Example Questions

Find:

* Customers who never placed an order
* Employees who never received a salary
* Students who never attended a course
* Users who never submitted a solution

All use the same pattern.

---

# Relationship Between LEFT JOIN and RIGHT JOIN

The instructor mentions:

```text
LEFT JOIN and RIGHT JOIN are essentially mirror images.
```

Example:

### Query 1

```sql
SELECT *
FROM users u
LEFT JOIN submissions s
ON u.user_id = s.user_id;
```

---

### Query 2

```sql
SELECT *
FROM submissions s
RIGHT JOIN users u
ON s.user_id = u.user_id;
```

Both produce the same result.

---

Visualization:

```text
A LEFT JOIN B

is equivalent to

B RIGHT JOIN A
```

---

# Full Outer Join

## Concept

A Full Outer Join combines:

```text
LEFT JOIN
+
RIGHT JOIN
```

into a single result.

---

# Why Do We Need It?

Consider:

### Users Table

Registered Users

| user_id |
| ------- |
| 101     |
| 102     |
| 103     |

---

### Submissions Table

Solution Submissions

| user_id |
| ------- |
| 101     |
| 102     |
| 999     |

Notice:

```text
999
```

is an unregistered user.

---

# INNER JOIN Result

```text
101
102
```

Only matching records.

---

# LEFT JOIN Result

```text
101
102
103
```

All registered users.

Missing:

```text
999
```

---

# RIGHT JOIN Result

```text
101
102
999
```

All submissions.

Missing:

```text
103
```

---

# FULL OUTER JOIN Result

```text
101
102
103
999
```

Everything from both tables.

---

# Hive Full Outer Join Syntax

```sql
SELECT *
FROM users u
FULL OUTER JOIN submissions s
ON u.user_id = s.user_id;
```

---

# Visualization

```text
Users

101
102
103

Submissions

101
102
999
```

---

Result:

| User ID |
| ------- |
| 101     |
| 102     |
| 103     |
| 999     |

---

# Why Is Full Outer Join Useful?

Useful when we need:

```text
Complete View of Data
```

including:

### Registered Users

and

### Unregistered Users

and

### Users With No Activity

all together.

---

# Hive vs SQL Note

The instructor mentions that Hive supports:

```sql
FULL OUTER JOIN
```

directly.

Some database systems historically lacked direct support and required combining:

```sql
LEFT JOIN
UNION
RIGHT JOIN
```

to simulate a Full Outer Join.

Hive provides it natively.

---

# Comparison of Join Types

| Join Type       | Returns                      |
| --------------- | ---------------------------- |
| INNER JOIN      | Only matching records        |
| LEFT JOIN       | All left + matching right    |
| RIGHT JOIN      | All right + matching left    |
| FULL OUTER JOIN | All records from both tables |

---

# Visual Memory Trick

```text
INNER JOIN

A ∩ B
(Common Only)
```

```text
LEFT JOIN

Entire A
+
Matching B
```

```text
RIGHT JOIN

Entire B
+
Matching A
```

```text
FULL OUTER JOIN

A ∪ B
(All Records)
```

# Interview Questions

### How do you find records present in one table but not another?

```sql
LEFT JOIN
+
WHERE column IS NULL
```

---

### Why is LEFT JOIN used to find inactive users?

Because unmatched rows produce NULL values that can be filtered.

---

### Difference between LEFT JOIN and FULL OUTER JOIN?

LEFT JOIN returns all records from the left table only.

FULL OUTER JOIN returns all records from both tables.

---

### Can LEFT JOIN and RIGHT JOIN return the same result?

Yes.

```sql
A LEFT JOIN B
```

is equivalent to

```sql
B RIGHT JOIN A
```

---

# Quick Revision

* LEFT JOIN keeps all records from the left table.
* Missing matches appear as NULL.
* `WHERE column IS NULL` helps find unmatched records.
* This pattern is widely used in SQL and Hive interviews.
* RIGHT JOIN is the mirror image of LEFT JOIN.
* FULL OUTER JOIN combines LEFT and RIGHT JOIN results.
* FULL OUTER JOIN returns every record from both tables.
* Hive supports FULL OUTER JOIN directly.

---

# One-Line Recall

**LEFT JOIN with `IS NULL` is the standard technique for finding missing or inactive records, while FULL OUTER JOIN combines all records from both tables regardless of whether matches exist.**
